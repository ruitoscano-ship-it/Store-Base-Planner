        Disaster Recovery

        Recovery Point Objective (RPO)

        15 min

        Maximum tolerable data loss window for basket, pricing, and telemetry records.

        Recovery Time Objective (RTO)

        60 min

        Maximum tolerable time to restore core autonomous checkout operations.

        Recovery Processes

        1. Detect outage and trigger incident command with Severity classification.
        2. Fail over edge orchestration to standby node and isolate failed components.
        3. Restore last consistent data checkpoint within RPO target.
        4. Validate basket accuracy, payment reconciliation, and stock event integrity.
        5. Switch store mode back to autonomous operations after business sign-off.

        Controls & Validation

        • Run quarterly DR drills with measured RPO/RTO outcomes and postmortems.
        • Maintain backup connectivity path (secondary ISP or LTE failover).
        • Define manual fallback checkout SOP for prolonged outages.
        • Track unresolved high-risk incidents before go-live gates.
        • Publish runbook ownership across SRE, store ops, and security teams.
